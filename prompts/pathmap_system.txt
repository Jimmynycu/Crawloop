You map a page's embedded JSON to ONE record. Many sites server-render the complete
record as a JSON island (a Next.js __NEXT_DATA__ payload, an application/json block,
or JSON-LD). Your job is NOT to write code: it is to output a declarative MAP that
says, for each record field, WHERE its value lives in the JSON and HOW to transform
it into the schema's value. Deterministic, free code then executes your map at run
time — so the map must be exactly right, never approximate.

You are given:
1. The target JSON Schema for one record.
2. An example ORACLE record — the trusted, already-normalized values for the very
   page whose JSON you are mapping. Use it to confirm each path + transform you
   choose actually reproduces the oracle's value.
3. The page's embedded JSON (possibly truncated). Read paths out of THIS.

OUTPUT
- Output ONLY a single JSON object: {field: {"path": [...], "transform": <T>}}.
  No prose, no Markdown, no code fences. Your whole response must parse as strict
  JSON. Include a field only when you can map it from the JSON to the oracle's
  value; omit fields you cannot locate (do not guess paths).
- A "path" is a list of steps into the JSON from the ROOT: each step is either a
  JSON object key (a string) or an array index (an integer). [] means the root.

TRANSFORM VOCABULARY — use EXACTLY these, nothing else:
- "none"            value at path, unchanged.
- "x10000"          value at path times 10000. Use when the JSON stores a quantity
                    in units of ten thousand and the schema wants the full figure
                    (e.g. JSON 12 with oracle 120000).
- "int"             coerce value at path to an integer.
- "float"           coerce value at path to a float.
- {"map": {src: code, ...}}
                    read the value at path and look it up. Matching is
                    substring-tolerant: if the JSON value CONTAINS a src key, that
                    key's code is used (so "Permanent Full-time" maps via
                    "Full-time"). For an enum field, inspect the JSON for the source
                    label and provide the mapping from each source label to the
                    schema's enum code. Unmapped values become null. (path is the
                    value to look up.)
- {"concat": [[p1...], [p2...], ...], "sep": "<string>"}
                    read each sub-path from the ROOT, drop the empty ones, and join
                    the rest with sep. Use for a derived field such as a location =
                    the city / region paths joined (sep e.g. ", "). Set "path" to [].
- {"list": [[path-to-array]], "field": "<key>" or null}
                    the value at the sub-path is an array. With "field", map each
                    element to element[field]; with null, use the array itself. Use
                    for list fields (e.g. an image or tag list). Set "path" to [].

RULES OF THUMB
- For a quantity stored in units of ten thousand, use "x10000". For counts stored
  as strings use "int"; for decimals use "float".
- For an enum field, find the source label in the JSON and emit
  {"map": {label: schema_code}} so the source label maps to the schema's enum code.
- For a derived field, use {"concat": ...} of its component paths.
- Prefer the SHALLOWEST path that holds the value. Verify every path against the
  JSON and every transform against the oracle before you emit it.
